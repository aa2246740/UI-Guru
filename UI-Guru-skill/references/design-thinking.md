## 设计思维核心

### 什么是设计思维？

设计思维是一种以人为本的问题解决方法，强调：

1. **用户优先**: 从用户需求出发
2. **迭代优化**: 快速原型，持续改进
3. **多维度思考**: 平衡功能、美学、技术
4. **数据驱动**: 基于反馈做决策

### 设计思维的 5 个阶段

```
共情 (Empathize) → 定义 (Define) → 构思 (Ideate) → 原型 (Prototype) → 测试 (Test)
     ↑                                                                    ↓
     └────────────────────────────────────────────────────────────────────┘
                              迭代循环
```

## 需求分析方法

### 阶段 1: 用户分析

#### 用户画像

**维度**:
- **人口统计**: 年龄、性别、职业、地区
- **技术水平**: 技术背景、使用习惯
- **使用场景**: 何时、何地、如何使用
- **痛点需求**: 现有解决方案的问题

**示例**:
```
用户：内容创作者
年龄：25-40岁
技术：中等，熟悉主流应用
场景：在办公室使用电脑，需要快速生成图片
痛点：现有工具复杂，生成结果不稳定
```

#### 使用场景分析

**5W1H 方法**:
- **Who**: 谁在使用？
- **When**: 什么时候使用？
- **Where**: 在哪里使用？
- **What**: 要完成什么任务？
- **Why**: 为什么要做这个任务？
- **How**: 如何完成任务？

### 阶段 2: 问题定义

#### 问题陈述模板

```
[用户] 需要 [需求]，因为 [原因]，
但 [当前障碍]，导致 [影响]。
```

**示例**:
```
内容创作者需要快速生成营销图片，因为时间紧迫，
但现有 AI 工具界面复杂，导致学习成本高、效率低。
```

#### 设计目标定义

**SMART 原则**:
- **Specific**: 具体明确
- **Measurable**: 可衡量
- **Achievable**: 可实现
- **Relevant**: 相关性强
- **Time-bound**: 有时限

**示例**:
```
目标：在 30 秒内完成一次图片生成请求，
包括输入提示词、选择风格、查看结果。
```

## 设计决策框架

### 决策维度 1: 产品定位

#### 产品类型分析

| 产品类型 | 设计特点 | 典型代表 |
|---------|---------|---------|
| B2C 应用 | 简洁、引导性强、视觉吸引 | 抖音、小红书 |
| B2B 应用 | 专业、高效、信息密度高 | Salesforce、钉钉 |
| 开发者工具 | 功能优先、可定制、深色系 | VS Code、GitHub |
| 企业应用 | 稳定、安全、一致性强 | 飞书、企业微信 |
| 创意工具 | 灵活、可视化、交互丰富 | Figma、Notion |

#### 设计风格选择

**极简风格**:
- 适用：内容型产品、年轻用户
- 特点：大量留白、突出核心内容
- 案例：Notion、Linear

**专业风格**:
- 适用：B2B、企业应用
- 特点：信息密度高、功能明确
- 案例：Salesforce、Slack

**创意风格**:
- 适用：创意工具、设计产品
- 特点：视觉丰富、交互有趣
- 案例：Figma、Canva

### 决策维度 2: 视觉方向

#### 色彩情绪分析

**冷色调** (蓝、青、紫):
- 情绪：专业、科技、冷静
- 适用：金融、医疗、企业应用
- 案例：PayPal、飞书

**暖色调** (橙、红、黄):
- 情绪：友好、温暖、活力
- 适用：社交、电商、娱乐
- 案例：抖音、淘宝

**中性色** (黑、白、灰):
- 情绪：极简、高级、百搭
- 适用：内容产品、设计工具
- 案例：Apple、Notion

#### 配色决策流程

```
1. 分析产品定位
   ↓
2. 确定品牌主色
   ↓
3. 选择辅助色 (2-3个)
   ↓
4. 定义功能色
   ↓
5. 建立中性色阶梯
   ↓
6. 验证对比度
```

### 决策维度 3: 交互模式

#### 交互密度选择

**低交互密度**:
- 适用：内容浏览、信息展示
- 特点：减少干扰，专注内容
- 案例：Medium、新闻网站

**中等交互密度**:
- 适用：大多数 Web 应用
- 特点：平衡功能和简洁
- 案例：ChatGPT、Gmail

**高交互密度**:
- 适用：专业工具、创意应用
- 特点：丰富交互，功能强大
- 案例：Figma、Photoshop

#### 动画使用原则

**用动画**:
- 状态反馈（加载、成功、错误）
- 引导注意力
- 增强愉悦感

**不用动画**:
- 纯信息展示
- 用户追求效率
- 性能敏感场景

## 常见陷阱规避

### 陷阱 1: 总是做纯黑/纯白界面

**问题原因**:
- 纯黑/纯白是最安全的选择
- AI 训练数据中这类设计较多
- 缺乏对产品的深入分析

**解决方法**:
1. **分析产品定位**
   - 企业应用 → 可以用深色
   - 消费产品 → 考虑浅色 + 彩色
   - 创意工具 → 可以用深色强调专业

2. **参考竞品配色**
   - 找 3-5 个同类产品
   - 分析它们的配色方案
   - 理解配色的原因

3. **建立色彩情绪板**
   - 根据品牌调性选择情绪
   - 用色彩传达情绪
   - 不要只用黑白

### 陷阱 2: 总是用紫色渐变

**问题原因**:
- 紫色渐变看起来"科技感"
- 在 AI 设计趋势中很流行
- 缺乏对品牌色彩的理解

**解决方法**:
1. **从品牌出发**
   - 品牌有标准色吗？
   - 品牌调性是什么？
   - 用品牌色作为主色

2. **行业色彩规范**
   - 金融：蓝、绿
   - 社交：品牌色
   - 医疗：青、绿
   - 电商：橙、红

3. **色彩心理学**
   - 蓝色：信任、专业
   - 绿色：健康、成长
   - 橙色：活力、行动
   - 红色：紧急、热情

### 陷阱 3: 过度设计

**问题表现**:
- 太多动画效果
- 太复杂的视觉
- 太多不必要的装饰

**解决方法**:
1. **设计减法**
   - 每个元素都有明确目的
   - 删除不必要的装饰
   - 留白也是设计

2. **用户测试**
   - 观察用户使用
   - 收集反馈
   - 迭代优化

3. **数据驱动**
   - 分析用户行为数据
   - 关注核心指标
   - 用数据验证设计

### 陷阱 4: 忽视可访问性

**问题表现**:
- 对比度不足
- 字体太小
- 缺乏键盘导航

**解决方法**:
1. **WCAG 标准**
   - 对比度至少 4.5:1
   - 支持键盘操作
   - 提供焦点指示

2. **测试工具**
   - 对比度检查器
   - 屏幕阅读器测试
   - 键盘导航测试

## 设计质量评估

### 视觉质量检查

**一致性**:
- [ ] 间距统一
- [ ] 颜色统一
- [ ] 字体统一
- [ ] 圆角统一

**层级清晰**:
- [ ] 信息层级明确
- [ ] 视觉焦点突出
- [ ] 次要信息弱化

**细节精致**:
- [ ] 对齐精确
- [ ] 边距合理
- [ ] 图标统一
- [ ] 阴影适度

### 功能质量检查

**交互完整**:
- [ ] 所有状态都有设计
- [ ] 交互反馈及时
- [ ] 错误处理完善

**性能优化**:
- [ ] 动画流畅
- [ ] 加载迅速
- [ ] 响应灵敏

**可访问性**:
- [ ] 对比度达标
- [ ] 支持键盘
- [ ] 语义清晰

### 用户体验检查

**易用性**:
- [ ] 操作直观
- [ ] 学习成本低
- [ ] 错误可恢复

**满意度**:
- [ ] 视觉愉悦
- [ ] 交互流畅
- [ ] 感受专业

## 单页面应用设计流程

### 阶段 1: 需求分析 (Requirements)

**目标**: 理解产品需求和用户场景

**关键问题**:
- 产品类型是什么？(Web/移动/桌面)
- 目标用户是谁？(B2C/B2B/开发者/企业用户)
- 行业特点是什么？(金融/医疗/社交/电商/工具)
- 品牌定位如何？(专业/亲和/创新/可靠/极简)
- 有哪些竞品可以参考？

**输出物**:
- 需求分析文档
- 竞品分析截图
- 用户场景描述

**示例**:
```
产品：AI 生图助手
类型：Web 应用
用户：内容创作者、营销人员
行业：AI 工具
品牌：简洁、高效、智能
竞品：ChatGPT、Midjourney、Stable Diffusion Web
```

---

### 阶段 2: 风格定位 (Style Positioning)

**目标**: 确定视觉方向，避免固定模板

**⚠️ 避免的陷阱**:
- ❌ 不要总是做纯黑/纯白界面
- ❌ 不要总是用紫色渐变
- ❌ 不要盲目跟风

**正确流程**:

1. **分析行业色彩特征**

| 行业 | 典型色彩 | 情绪传达 |
|------|---------|---------|
| 金融 | 蓝色、绿色 | 信任、稳定 |
| 社交 | 品牌色 | 友好、活力 |
| 开发者工具 | 深色系 | 专业、专注 |
| 电商 | 橙色、红色 | 行动、紧迫 |
| 医疗 | 青色、绿色 | 健康、平静 |
| 企业应用 | 蓝色、灰色 | 专业、可靠 |

2. **建立色彩情绪板**
   - 主色：用于核心交互（按钮、链接、激活状态）
   - 辅助色：用于信息层级
   - 功能色：成功、警告、错误
   - 中性色：背景、边框、文字

3. **确定设计令牌**
   ```css
   --color-primary: #0071e3;
   --color-background: #f5f5f7;
   --color-card: #ffffff;
   --color-text: #1d1d1f;
   --color-text-secondary: #6e6e73;
   ```

**输出物**:
- 配色方案
- 设计令牌 (tokens.md)

---

### 阶段 3: 设计系统定义 (Design System)

**目标**: 建立一致的设计基础

#### 字体系统

**中文产品**:
- 思源黑体 / 苹方 / 阿里巴巴普惠体

**英文产品**:
- Inter / SF Pro / Roboto

**字号层级**:
```
12px - 辅助信息
13px - 次要文字
14px - 正文
15px - 输入框
16px - 小标题
18px - 页面标题
24px - 特大标题
```

#### 间距系统

**4px 基础单位**:
```
4px  - 最小间距
8px  - 紧凑间距
12px - 标准间距
16px - 舒适间距
20px - 宽松间距
24px - 大间距
```

#### 圆角系统

```
4px  - 气泡指向角
8px  - 小按钮、标签
12px - 卡片
16px - 大按钮、输入框
24px - 丸型按钮
50%  - 圆形元素
```

**输出物**:
- tokens.md
- 设计系统文档

### 阶段 4: 组件设计 (Component Design)

**目标**: 按需设计组件，不是使用模板

**设计原则**:
1. 根据功能需求设计组件
2. 定义组件的所有状态
3. 考虑组件的组合方式
4. 编写完整的文档

**组件分类**:

1. **布局组件**
   - Container / Wrapper
   - Header / Sidebar / Footer
   - Grid / Flex 布局

2. **反馈组件**
   - Button / Toggle / Checkbox
   - Input / Select / Textarea
   - Modal / Tooltip / Toast

3. **展示组件**
   - Card / List / Table
   - Avatar / Badge / Tag
   - Chart / Image / Video

4. **导航组件**
   - Menu / Breadcrumb / Tabs
   - Pagination / Sidebar

**输出物**:
- components.md
- 组件代码

### 阶段 5: 交互与动画 (Interactions)

**目标**: 定义交互细节和动画效果

#### 动画时长

```
0.1s - 微交互（悬停、点击）
0.2s - 快速过渡
0.3s - 标准动画
0.4s - 中等动画
0.5s+ - 缓慢动画（谨慎使用）
```

#### 缓动函数

```
ease-out - 退出动画（推荐）
ease-in-out - 往返动画
ease-in - 进入动画（少用）
linear - 循环动画
cubic-bezier - 弹性效果
```

#### 交互状态

每个可交互元素需要定义：
- Default（默认）
- Hover（悬停）
- Active/Focus（激活）
- Disabled（禁用）
- Loading（加载中）
- Error（错误）

**输出物**:
- interactions.md

### 阶段 6: 原型开发 (Prototyping)

**目标**: 创建可交互的原型，验证设计

#### 交付物 1: interactive.html

**功能**:
- 完整的用户流程
- 所有交互状态
- 场景演示
- 可直接在浏览器打开

**结构**:
```html
<!DOCTYPE html>
<html>
<head>
    <style>
        /* CSS 变量 */
        /* 全局样式 */
        /* 组件样式 */
        /* 动画定义 */
    </style>
</head>
<body>
    <!-- 页面结构 -->
    <!-- 交互脚本 -->
</body>
</html>
```

#### 交付物 2: design-guide.html

**布局**:
- 左侧: 组件导航
- 中间: 实时预览
- 右侧: 规范 + 代码

**功能**:
- 悬停显示尺寸标注
- 一键复制代码
- 快速跳转到原型

**输出物**:
- interactive.html
- design-guide.html

## 设计评审与迭代

### 评审检查点

**视觉检查**:
- [ ] 设计一致性
- [ ] 间距统一性
- [ ] 颜色对比度
- [ ] 字体层级

**功能检查**:
- [ ] 所有交互状态
- [ ] 动画流畅性
- [ ] 响应式适配
- [ ] 可访问性

**文档检查**:
- [ ] 代码完整性
- [ ] 文档清晰度
- [ ] 示例可用性

### 迭代原则

1. **小步快跑**: 每次改动一个维度
2. **用户反馈**: 优先解决用户痛点
3. **保持一致**: 不破坏已有设计
4. **文档同步**: 及时更新文档

## 常见问题处理

### Q1: 如何避免总是做纯黑/纯白界面？

**分析产品定位**:
- 企业应用 → 深色/灰色系
- 消费产品 → 浅色/彩色系
- 开发者工具 → 深色系
- 社交应用 → 品牌彩色

**参考竞品**: 找 3-5 个同类产品，分析它们的配色方案

### Q2: 如何选择合适的配色？

**步骤**:
1. 确定品牌主色
2. 选择 2-3 个辅助色
3. 定义功能色（成功、警告、错误）
4. 建立中性色阶梯

**工具**:
- Coolors.co
- Adobe Color
- Material Design Color Tool

### Q3: 动画应该多快？

**原则**:
- 微交互（悬停、点击）: 0.1-0.2s
- 状态切换: 0.2-0.3s
- 进场动画: 0.3-0.4s
- 复杂动画: 0.4-0.5s

**避免**: 超过 0.5s 的动画（除非特殊场景）

### Q4: 如何确保设计一致性？

**方法**:
1. 建立设计令牌系统
2. 使用 CSS 变量
3. 创建组件库
4. 编写设计规范
5. 定期设计评审
